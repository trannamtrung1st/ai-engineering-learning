# We Event UI/UX Docs

This folder contains the detailed UI/UX documentation split by concern so design, product, and engineering can review independently and implement in parallel.

## Document map

- `00-production-ui-quality-bar.md`  
  **Harness entry requirement.** Minimum bar for production UI—not demo scaffolds.

- `01-design-overview.md`  
  Product intent, users, UX principles, and BRD alignment boundaries.

- `02-ui-framework-tech-stack.md`  
  Recommended implementation stack and technical UX constraints for the web MVP.

- `03-design-system-basics.md`  
  Core system foundations (grid, typography behavior, interaction conventions, responsive rules).

- `04-design-tokens.md`  
  Semantic token model for color, typography, spacing, radius, and motion.

- `05-common-ui-components.md`  
  Shared component standards and behavior contracts.

- `06-app-layout-components.md`  
  App shell patterns and layout conventions by role.

- `07-event-specific-components.md`  
  Event domain components (capacity, waitlist, check-in, eligibility, audit).

- `08-forms-validation-ux.md`  
  Form composition, validation strategy, and error messaging UX.

- `09-page-list.md`  
  Sitemap by role and key page responsibilities.

- `10-user-flows.md`  
  End-to-end flows with decisions, edge cases, and state transitions.

- `11-wireframes.md`  
  Required wireframes, content blocks, and annotation standards.

- `12-ui-states.md`  
  Data/action/domain states and rendering guidance.

- `13-accessibility-basics.md`  
  MVP accessibility baseline and testing checklist.

## Scope and traceability

- Source requirements: `docs/brds`.
- Scope target: web MVP only.
- Out-of-scope items (payments, native app, advanced external integrations) are intentionally excluded from UI behavior definitions.
