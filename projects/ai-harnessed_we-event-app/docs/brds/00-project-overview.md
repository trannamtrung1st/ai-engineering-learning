# We Event BRD - Project Overview

## 1. Project Overview
We Event is a centralized event management platform for educational environments (schools, clubs, departments, training organizations). It supports the full event lifecycle: event creation, registration, capacity and waitlist management, check-in, feedback collection, and participation certificate eligibility evaluation.

The product targets two primary user groups:
- Participants (students/learners): discover events, register, track status, check in, submit feedback.
- Organizers: manage operations through a standardized workflow, reducing manual work and data errors.

## 2. Business Context
A common current state is operating events with fragmented tools (Google Forms, Excel, Zalo, email), which leads to:
- Duplicate registrations and poor control of repeated signups.
- Over-capacity enrollment due to lack of real-time capacity control.
- Non-transparent waitlists that are hard to prioritize and convert.
- Slow check-in and inconsistent attendance data.
- Difficulty determining certificate-eligible participants.
- Difficulty consolidating event outcomes for quality improvement.

We Event is built to replace manual steps with a single, traceable, reportable system.

## 3. Objectives
### 3.1 Business Objectives
- Standardize the end-to-end event execution process.
- Minimize dependency on fragmented tools and manual operations.
- Improve transparency in capacity, waitlist, attendance, and certificate management.
- Improve participant registration and attendance experience.

### 3.2 Operational Objectives
- Reduce post-event time spent consolidating registration and attendance lists.
- Reduce data mismatches between registrations and actual attendance.
- Control overbooking risk through defined business rules.

### 3.3 Success Signals (Business Level)
- Platform adoption increases over semesters compared with legacy workflows.
- Valid registration ratio remains high and stable.
- In-window check-in success rate increases.
- Organizers can export certificate-eligible participant lists without manual processing.
