# Playwright UI Regression — Browser Test Codegen

Committed Playwright specs for UI regression, generated by the browser test agent after each full verification pass.

## Relationship to other test layers

| Layer | Location | Mechanism |
| --- | --- | --- |
| API e2e | `tests/e2e` | HTTP/inject — business flows |
| Browser gate | Playwright MCP | Agent-driven verification per slice |
| **Playwright UI regression** | `tests/playwright-ui/scenarios/` | Committed specs — `npm run test:playwright-ui` |

TestGen emits `layer: browser` **JSON specs** only (`docs/test-cases/items/`). Executable Playwright UI tests are owned by the browser test agent codegen step.

## Workspace layout

```
tests/playwright-ui/
  package.json
  playwright.config.ts
  src/support/
    auth.ts          # dev login helpers (customize per product)
    constants.ts     # ports, default passwords
  scenarios/
    <slice-id>.spec.ts   # written/updated by browser tester
```

Index: `ai-harness/playwright-regression-index.json`

## When codegen runs

After the **full verification phase** completes (whether or not all `TC-*` cases passed — capture exercised flows). Skip codegen for flows marked `SKIP — physical-device` only.

The browser test agent:

1. Finishes `TC-*` checklist + UX audit
2. Writes/updates `tests/playwright-ui/scenarios/<slice-id>.spec.ts`
3. Emits `playwright-regression: tests/playwright-ui/scenarios/<slice-id>.spec.ts (N tests)` in output

## Spec template

```typescript
import { test, expect } from "@playwright/test";
import { loginViaApi } from "../src/support/auth.js";
import { WEB_BASE_URL } from "../src/support/constants.js";

/**
 * Slice: <slice-id>
 * Source TCs: TC-<tag>-NNN, ...
 * Generated by browser test agent — do not hand-edit without re-running gate.
 */
test.describe("<slice-id>", () => {
  test.beforeEach(async ({ page }) => {
    await loginViaApi(page, process.env.PLAYWRIGHT_USER_EMAIL!, process.env.PLAYWRIGHT_USER_PASSWORD!);
  });

  test("primary flow from TC-* checklist", async ({ page }) => {
    await page.goto(`${WEB_BASE_URL}/your-route`);
    await expect(page.getByRole("heading", { name: /expected title/i })).toBeVisible();
  });
});
```

## Selector conventions

1. Prefer `data-testid` when present in the app
2. Else `getByRole`, `getByLabel`, `getByText` (product copy)
3. Avoid brittle CSS class selectors
4. Use `WEB_BASE_URL` from constants (`http://localhost:3007` in preview by default)

## Auth helpers

Use dev login flow documented in `docs/technical/10-local-development-setup.md`. Shared helpers live in `src/support/auth.ts` — customize per product roles and routes.

## Idempotent updates

When `<slice-id>.spec.ts` already exists:

- Merge new tests for newly exercised `TC-*` flows
- Update selectors when UI changed
- Remove obsolete tests only when the flow no longer exists in acceptance tags
- Do not duplicate `test()` blocks for the same `TC-*` id

## Running regression

```bash
npm run aih:preview          # start preview stack
npm run aih:preview:verify   # confirm health
npm run test:playwright-ui   # run all UI regression specs (full suite)
```

### Slice-scoped checks (`aih:check`)

When `computationalChecks.playwrightScope` is `"slice"` (default), `npm run aih:check -- <sliceId>` runs only that slice's spec via `npx playwright test <spec>` — not the full workspace suite. Full regression is reserved for `playwrightScope: "full"`, manual `npm run test:playwright-ui`, or acceptance-suite slices.

### Case budget

`browserTest.playwrightMaxCasesPerSlice` in `ralph-loop.json` caps how many browser cases the tester prioritizes per slice (P0/P1 first when the checklist exceeds the budget). Committed Playwright specs may still cover more flows over time.

Optional gate in `ralph-loop.json` — `test:playwright-ui` is optional until `playwright-ui-workspace` slice passes.

## Deterministic test hooks

CI cannot use physical cameras, live GPS, or other hardware-dependent APIs. Approved patterns for committed specs:

| Hook | Purpose |
| --- | --- |
| `window.__PRODUCT_TEST__` namespace | Inject inputs without hardware (e.g. `simulateQrDecode(raw)`) |
| `page.addInitScript` | Stub `navigator.geolocation`, `permissions.query`, `getUserMedia` |
| `page.route('**/api/...')` | Stub API responses for outcome paths |
| `loginViaApi(page, …)` | Fast auth via API instead of form login |

Expose test hooks only behind a `window` namespace cleared on unmount — do not ship production behavior that depends on them.

See [11-testing-plan.md](../../docs/technical/11-testing-plan.md) §7.3 for product-specific examples.

## Output signal

```
playwright-regression: tests/playwright-ui/scenarios/<slice-id>.spec.ts (4 tests)
```

Harness parses this line into `*-browser-test.json` → `playwrightSpec`, `playwrightTestCount`.
