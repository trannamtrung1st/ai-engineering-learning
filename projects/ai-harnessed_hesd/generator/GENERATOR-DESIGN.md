# Generator Design

Concise index for the portable spec generator. Independent from `ai-harness/` and from generated repo artifacts.

## Component map

| Component | Location |
|---|---|
| Step queue | `steps-backlog.json` |
| Loop policy | `workflows/gen-loop.json` |
| Models | `config/models.json`, env `GEN_MODEL` |
| Doc outlines | `config/doc-outlines.json` |
| Agents | `agents/doc-writer.prompt.md`, `harness-planner.prompt.md`, `doc-reviewer.prompt.md` |
| Internal state | `state/progress.md`, `state/guardrails.md` |
| Templates | `templates/ai-harness/` (vendored static harness) |
| Validation | `scripts/verify-step.sh` and validators |
| Observability | `generated/runs/<timestamp>-*.txt` |

## Loop flow

```
pick step → [scaffold|agent|gate]
  → agent writes outputs (or scaffold script copies templates)
  → verify-step (structure, IDs, crossrefs, harness JSON, no generator refs)
  → optional doc-reviewer (REVIEW_PASS)
  → mark passes → commit → repeat
```

Scripts: `generate.sh` (entry), `gen-loop.sh` (autonomous), `gen-once.sh` (single step).

## Phases

| Phase | Steps | Output |
|---|---|---|
| 0 | input-validate | `docs/product-meta.json` |
| 1 | BRD (6 steps) | `docs/brds/*.md` |
| 2 | Technical (5 steps) | `docs/technical/*.md` |
| 3 | UI/UX (4 steps) | `docs/ui-ux/*.md` |
| 4 | Harness (4 steps) | `ai-harness/` |
| 5 | Bootstrap | `package.json`, `.gitignore` |

## Signals

| Signal | Meaning |
|---|---|
| `STEP_DONE <id>` | Agent finished |
| `STEP_BLOCKED <reason>` | Agent blocked |
| `REVIEW_PASS` / `REVIEW_FAIL` | Doc review |
| `GEN_COMPLETE` | All steps passed |

## Independence

- Zero runtime imports from `ai-harness/scripts/`
- Generated repo artifacts (`docs/`, `ai-harness/`, root `package.json`) must **not** reference `generator/`
- Product metadata lives at `docs/product-meta.json` (project-owned, not under `generator/`)
- Vendored harness in `templates/ai-harness/` copied on `harness-scaffold` step
- Dynamic harness files generated by `harness-planner` agent
- HESD (`projects/ai-harnessed_hesd/`) is the reference implementation for integration/e2e testing-pattern sections in `docs/technical/11-testing-plan.md`
- Run from `generator/` via `npm run gen:loop`; target repo via `GEN_REPO_ROOT` when generator lives elsewhere
