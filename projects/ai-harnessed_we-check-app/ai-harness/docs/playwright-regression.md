# Playwright UI Regression — Browser Test Codegen

Committed Playwright specs for UI regression, generated by the browser test agent after each full verification pass.

## Relationship to other test layers

| Layer | Location | Mechanism |
| --- | --- | --- |
| API e2e | `tests/e2e` | Fastify `inject` — business flows A–D |
| Browser gate | Playwright MCP | Agent-driven verification per slice |
| **Playwright UI regression** | `tests/playwright-ui/scenarios/` | Committed specs — `npm run test:playwright-ui` |

TestGen emits `layer: browser` **JSON specs** only (`docs/test-cases/items/`). Executable Playwright UI tests are owned by the browser test agent codegen step.

## Workspace layout

```
tests/playwright-ui/
  package.json
  playwright.config.ts
  src/support/
    auth.ts          # dev login helpers
    constants.ts     # ports, default passwords
  scenarios/
    <slice-id>.spec.ts   # written/updated by browser tester
```

Index: `ai-harness/playwright-regression-index.json`

## When codegen runs

After the **full verification phase** completes (whether or not all `TC-*` cases passed — capture exercised flows). Skip codegen for flows marked `SKIP — physical-device` only.

The browser test agent:

1. Finishes `TC-*` checklist + UX audit
2. Writes/updates `tests/playwright-ui/scenarios/<slice-id>.spec.ts`
3. Emits `playwright-regression: tests/playwright-ui/scenarios/<slice-id>.spec.ts (N tests)` in output

## Spec template

```typescript
import { test, expect } from "@playwright/test";
import { loginAsStudent } from "../src/support/auth.js";
import { WEB_BASE_URL } from "../src/support/constants.js";

/**
 * Slice: web-student-checkin
 * Source TCs: TC-AC-07-013, TC-AC-08f-002
 * Generated by browser test agent — do not hand-edit without re-running gate.
 */
test.describe("web-student-checkin", () => {
  test.beforeEach(async ({ page }) => {
    await loginAsStudent(page);
  });

  // TC-AC-07-013: preflight gate shows session summary before GPS
  test("preflight gate shows session summary", async ({ page }) => {
    await page.goto(`${WEB_BASE_URL}/check-in?token=...`);
    await expect(page.getByRole("heading", { name: /điểm danh/i })).toBeVisible();
  });
});
```

## Selector conventions

1. Prefer `data-testid` when present in the app
2. Else `getByRole`, `getByLabel`, `getByText` (Vietnamese copy)
3. Avoid brittle CSS class selectors
4. Use `WEB_BASE_URL` from constants (`http://localhost:3007` in preview)

## Auth helpers

Use dev login flow documented in `docs/technical/10-local-development-setup.md`. Shared helpers live in `src/support/auth.ts`.

## Idempotent updates

When `<slice-id>.spec.ts` already exists:

- Merge new tests for newly exercised `TC-*` flows
- Update selectors when UI changed
- Remove obsolete tests only when the flow no longer exists in acceptance tags
- Do not duplicate `test()` blocks for the same `TC-*` id

## Running regression

```bash
npm run aih:preview          # start preview stack
npm run aih:preview:verify   # confirm health
npm run test:playwright-ui   # run all UI regression specs
```

Optional gate in `ralph-loop.json` — `test:playwright-ui` is optional until `playwright-ui-workspace` slice passes.

## Output signal

```
playwright-regression: tests/playwright-ui/scenarios/web-student-checkin.spec.ts (4 tests)
```

Harness parses this line into `*-browser-test.json` → `playwrightSpec`, `playwrightTestCount`.
