## HESD Attendance — Local Setup

This repository bootstraps the HESD workshop attendance product from the official [Next.js + Supabase starter](https://github.com/vercel/next.js/tree/canary/examples/with-supabase).

### Environment variables

Copy `.env.example` to `.env.local` (web) and configure values from your [Supabase project API settings](https://supabase.com/dashboard/project/_/settings/api):

| Variable | Scope | Notes |
| --- | --- | --- |
| `NEXT_PUBLIC_SUPABASE_URL` | web | Supabase project URL |
| `NEXT_PUBLIC_SUPABASE_PUBLISHABLE_KEY` | web | Publishable or legacy anon key |
| `NEXT_PUBLIC_API_URL` | web | NestJS API base URL (`http://localhost:3001`) |
| `DATABASE_URL` | **api only** | Postgres URL for Drizzle (`postgresql://hesd:hesd_dev@localhost:5432/hesd` with compose) |
| `SUPABASE_SERVICE_ROLE_KEY` | **api only** | Never expose to the browser |
| `SUPABASE_JWT_SECRET` | **api only** | Local JWT validation; hosted uses JWKS (Story 1.3) |
| `PORT` | api | API listen port (default `3001`) |
| `WEB_ORIGIN` | api | CORS origin for web (`http://localhost:3000`) |

**Key naming:** The template uses `NEXT_PUBLIC_SUPABASE_PUBLISHABLE_KEY`. Supabase dashboard may still label this as `anon` key — the value is interchangeable during the transition.

**Security:** Never put `SUPABASE_SERVICE_ROLE_KEY`, `DATABASE_URL`, or `SUPABASE_JWT_SECRET` in `NEXT_PUBLIC_*` variables. The web app does not import Drizzle.

### Run locally (split stack)

**Terminal 1 — Postgres + API (hot-reload):**

```bash
docker compose --profile local up
```

API health check: [http://localhost:3001/api/v1/health](http://localhost:3001/api/v1/health)

**Terminal 2 — Next.js web:**

```bash
npm install
npm run dev
```

Open [http://localhost:3000](http://localhost:3000). Role placeholder routes:

- `/admin` — admin shell (Story 1.5+)
- `/instructor` — instructor shell
- `/student` — student mobile shell

**Integration profile (CI migrate + seed):**

```bash
docker compose --profile integration up
```

The `migrate` service runs `db:migrate` and `db:seed` once, then exits.

### Web-only quick start

```bash
npm install
npm run dev
```

Auth routes from the starter remain under `/auth/*`.

## Features

- Works across the entire [Next.js](https://nextjs.org) stack
  - App Router
  - Pages Router
  - Proxy
  - Client
  - Server
  - It just works!
- supabase-ssr. A package to configure Supabase Auth to use cookies
- Password-based authentication block installed via the [Supabase UI Library](https://supabase.com/ui/docs/nextjs/password-based-auth)
- Styling with [Tailwind CSS](https://tailwindcss.com)
- Components with [shadcn/ui](https://ui.shadcn.com/)
- Optional deployment with [Supabase Vercel Integration and Vercel deploy](#deploy-your-own)
  - Environment variables automatically assigned to Vercel project

## Clone and run locally

1. You'll first need a Supabase project which can be made [via the Supabase dashboard](https://database.new)

2. Create a Next.js app using the Supabase Starter template npx command

   ```bash
   npx create-next-app --example with-supabase with-supabase-app
   ```

   ```bash
   yarn create next-app --example with-supabase with-supabase-app
   ```

   ```bash
   pnpm create next-app --example with-supabase with-supabase-app
   ```

3. Use `cd` to change into the app's directory

   ```bash
   cd with-supabase-app
   ```

4. Rename `.env.example` to `.env.local` and update the following:

  ```env
  NEXT_PUBLIC_SUPABASE_URL=[INSERT SUPABASE PROJECT URL]
  NEXT_PUBLIC_SUPABASE_PUBLISHABLE_KEY=[INSERT SUPABASE PROJECT API PUBLISHABLE OR ANON KEY]
  ```
  > [!NOTE]
  > This example uses `NEXT_PUBLIC_SUPABASE_PUBLISHABLE_KEY`, which refers to Supabase's new **publishable** key format.
  > Both legacy **anon** keys and new **publishable** keys can be used with this variable name during the transition period. Supabase's dashboard may show `NEXT_PUBLIC_SUPABASE_ANON_KEY`; its value can be used in this example.
  > See the [full announcement](https://github.com/orgs/supabase/discussions/29260) for more information.

  Both `NEXT_PUBLIC_SUPABASE_URL` and `NEXT_PUBLIC_SUPABASE_PUBLISHABLE_KEY` can be found in [your Supabase project's API settings](https://supabase.com/dashboard/project/_?showConnect=true)

5. You can now run the Next.js local development server:

   ```bash
   npm run dev
   ```

   The starter kit should now be running on [localhost:3000](http://localhost:3000/).

6. This template comes with the default shadcn/ui style initialized. If you instead want other ui.shadcn styles, delete `components.json` and [re-install shadcn/ui](https://ui.shadcn.com/docs/installation/next)

> Check out [the docs for Local Development](https://supabase.com/docs/guides/getting-started/local-development) to also run Supabase locally.
